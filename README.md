# intl-28loop-guide

本仓库用于归档和发布多个独立 HTML 页面。页面内容涵盖不同主题，彼此之间没有功能依赖，适合作为静态页面集合进行浏览或引用。

## 项目简介

`intl-28loop-guide` 是一个以 HTML 页面为主要存储形式的归档仓库。每个页面均为独立文件，无需后端服务即可在浏览器中直接打开查看。仓库本身不绑定任何特定域名或网站，仅提供页面文件的托管与版本管理。

## 目录说明

```
.
├── README.md          # 本说明文件
├── index.html         # 仓库入口页面（如有）
├── pages/             # 存放各独立 HTML 页面
│   ├── page-01.html
│   ├── page-02.html
│   └── ...
└── assets/            # 页面所需的公共资源（如 CSS、图片等）
    ├── styles.css
    └── images/
```

- **pages/**：核心目录，所有独立 HTML 页面均存放于此。
- **assets/**：存放页面引用的静态资源文件，便于统一管理。
- **index.html**：可选入口页面，可提供页面列表或导航。

## 页面归档说明

- 每个页面均为完整的、自包含的 HTML 文件，可直接在浏览器中运行。
- 页面之间无交叉引用，不依赖外部服务或数据库。
- 页面内容可能随时间更新，历史版本可通过 Git 提交记录追溯。

## 维护说明

- 欢迎通过 Issue 提交页面内容建议或问题反馈。
- 如需新增页面，请将 HTML 文件放入 `pages/` 目录，并保持文件命名清晰。
- 请勿在页面中嵌入任何形式的追踪代码、广告或恶意脚本。
- 本仓库仅用于归档与展示，不对页面内容的准确性或适用性作任何保证。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 在浏览器中直接打开 `pages/` 目录下的任意 HTML 文件。
3. 也可通过 GitHub Pages 等静态托管服务部署后在线访问（需自行配置）。
